Introduction

Dashboard parameters allow a dashboard editor to add custom parameters to the dashboard. There are two types of parameters available:

  • General Parameters: Are not bound to a single widget(Widget column is empty in the add parameter window). These parameters can be used to substitute placeholders(e.g "$param_name$") or to change settings over multiple widget types.
  • Widget Settings Parameters:  For every widget setting there is a default parameter. These are useful to globally override the setting for the widget.

There are two modes for the parameters. Some parameters will only allow a single mode:

  • Global: Will globally override the specific widget setting.
  • Substitute:  Will substitute placeholders(e.g "$param_name$") only at the settings where the placeholders are manually selected or entered.

Adding Parameters to a Dashboard

To add a parameter to a dashboard:

  • Add Parameter:  Go to "Edit Mode >> Parameters >> Add Parameter" and click on one of the parameters to add it to the list of parameters.
  • Naming: Name your parameter, blanks are not allowed, use underscore instead. The name will be used to create the label in the Parameters Widget. For example the name "my_parameter" will result in the label "My Parameter"(underscores replaced with blanks, words capitalized).
  • Configure:  Choose the mode and set the default value of the parameter and click save.
  • Add to Widget Setting:  If your parameter has the mode Substitute, open the settings of a widget and enter the placeholder(e.g "$my_parameter$") into a text field or select it from a selection box.
  • Provide to Dashboard User:  To provide the parameter to the user of the dashboard, see the next section.

The Parameter Widget

Dashboard users can be enabled to change specific dashboard parameters by using the Parameters Widget found in the Standard Widgets add " Standard Widgets >> Parameters ". You can add multiple Parameter widgets to a single dashboard, but every parameter should only be chosen once to be displayed as a parameter, else it is not predictable which of the two parameters will actually be used. When a user clicks the Update button, the parameters are stored in the browser cache and added to the URLs as parameters.
The order of the input fields is defined by the time the parameters were created (oldest first).

Parameters Hierarchy

To properly work with parameters, you should understand how the priorities between the various sources of the parameter values are:

  1. URL Parameters:  URL Parameters will override the parameters in the browsers cache. This enables you to create a single dashboard and create different links/bookmarks that will filter by specific data.
  2. Browser Cache Parameters:  The Parameters from the browser cache will be merged with the default values of the parameters. If a user has set a custom value for a parameter, it will override the default value. If a parameter does not have a custom value, the default value will be used.
  3. Default Values:  The default values are set in the parameters edit window. These will be used if the user has not set custom values.

Limitations

The following are the limitations you have when working with parameters:

  1. Error Prone:  By using parameters you start entering the field of development. This means you can introduce errors. For example if you define a parameter for a select (e.g. $chart_type$) and you rename or delete the parameter afterwards, the widget will not be able to load correctly as the parameter does not exists and the placeholder doesn't get replaced.
  2. Same Settings on different Widget Types:  Same Settings on different widget types(e.g. Example Mode) cannot be globally overridden with a parameter that is bound to a certain widget type. Some Widget implementations might provide special parameters that will span over multiple widget types for the same setting.
  3. Replica Widget:  The Replica widget will not copy the parameters from the source dashboard. If the copied widget contains parameters, the parameters have also to be defined on the target dashboard.
