The metadata command is used to add custom properties to a query.

The following properties will also be used by the query engine itself:

  • name:  Will be used to give the current query a name. Used to identify the query with other commands like 'resultcompare'.
  • title:  When set to true, the name will be shown as a title on the User Interface. Title will also contain the queries execution duration.
  • description:  Will be used as a description on the user interface.
  • template:  Set to true if query is a template to be used with mimic command. Query will be cancelled at the metadata command. Any results caused by the query will be removed.

Following code has two queries and adds the properties name and description to each query. The name property will also be used in the user interface for the title of the query. 

| metadata 
	name="Random Records" 
	title=true
	description="Query creates random records." 
	custom="whatever you wanna say."
| source random
;
| metadata 
	name="More Randomness" 
	title=true
	description="just another example query"
| source random
