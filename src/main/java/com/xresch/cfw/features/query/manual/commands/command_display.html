This command lets you change how the data is rendered. By default data is rendered as a table. With the 'as' parameter you can choose the renderer that will be used to display the data.

Panels

The following query creates 42 random records and displays them as panels. 

source random records = 42
| display as=panels

Sticky Table Headers

The following will render the data as a table, and make the header of the table stick when scrolling and keep it visble. 

| source random 
| display sticky = true

Sorting: Basic

To enable sorting, you can define the fieldnames you want to sort by as an array: 

| source random records= 1000
| display
	sortoptions = [FIRSTNAME, LASTNAME]

Sorting: Advanced

To do advanced sorting, you can define an object like the following: 

| source random records= 1000
| display
	sortoptions = { 
		  "Firstname": [ [FIRSTNAME], ['asc'] ] 
		, "Lastname": [ [LASTNAME], ['asc'] ] 
		, "First and Lastname": [ [FIRSTNAME, LASTNAME], ['asc', 'asc'] ] 
		, "First and Location": [ [FIRSTNAME, LOCATION], ['asc', 'desc'] ] 
	}

Visible Fields

Further you can set the fields that should be visible by providing an array of fieldnames. This might not work for all renderers, for example the JSON display will always show all fields. 

| source random
| rename URL = url
| rename LAST_LOGIN = "Last Login"
| display as=table  visiblefields=[FIRSTNAME, LASTNAME, "Last Login", url]

Define Title

For renderers like panels, cards and tiles, you can set the fields used for the title by setting the parameters titlefields and titleformat. If titleformat is not set, the field values are joined with a blank as a separator: 

| source random
| rename URL = url
| rename LAST_LOGIN = "Last Login"
| display as=panels  
    visiblefields=[FIRSTNAME, LASTNAME, "Last Login", url] 
    titlefields=[INDEX, FIRSTNAME, LASTNAME, LOCATION]
    titleformat="{0}: {2} {1} (Location: {3})"

Colored Status Map

The statusmap renderer will only properly display when the menu is turned off using menu=false. The following example does that: 

| source random records=50
| remove _source
| formatrecord 
	[(VALUE >= 80), "cfw-red"] 
	[(VALUE >= 40), "cfw-yellow"] 
	[(VALUE >= 0), "cfw-green"] 
| sort VALUE reverse=true
| display as=statusmap menu=false

Download and Store

You can display a button in the menu to give the option to download the data in either CSV, JSON or XML format. You can also display a store button that allows the user to store their data in the File Manager: 

| source random
| display as=table
	download = true
	store = true
	# menu=button # above buttons do not work when the menu is displayed as a button
