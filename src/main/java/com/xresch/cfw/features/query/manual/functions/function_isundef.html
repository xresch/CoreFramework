This function will return true only if the specified fieldname is undefined for the record. This is different than checking for null, as null is assigned to a fieldname.

The following query creates sample records and show some uses of the iUndef-function:

| record
	[NAME, VALUE]
	["Aurora", 42]
	["Treya"]
	["Marion", "Panna Cotta"]
	["Alejandra"]
	[null, null]
| set eval = isUndef(VALUE)
