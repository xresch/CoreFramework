You can store your storedQuery and access information in a central place under Menu >> Tools >> StoredQuery. These storedQuery can then be accessed in the query language using the function storedQuery(name). As passwords, tokens and such will be needed in plain text when used in the query, everyone having access to a specific query will be able to read out the actual unencrypted values.

This can also be useful to not have a URL distributed in 100 queries over 25 dashboards, but having it defined in a single place and easy to update in case it ever will change.
