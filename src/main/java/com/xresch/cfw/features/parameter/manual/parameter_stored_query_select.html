This parameter allows you to execute a stored query and use the resulting records as values for a selection box. When the data returned by the query has a single column, the values will be use as the value and the label for the select options. If the data contains two columns, the first will be used as the value and the second as the label.

When using this parameter with a query, the value of the parameter will be a simple string. For example, following is the stored query we define for the values for a parameter 'name_filter': 

| execute earliestSet(null, -1, "d") # set timeframe here else default last 30 minutes will be used
| source random
| keep FIRSTNAME
| distinct FIRSTNAME
| sort FIRSTNAME

Below query would get a string as the parameter:

| paramdefaults
	name_filter = "Aurora" 	# Aurora is default, a string will be taken from parameter
