The default application contains some functionality to analyze the behaviour and status of the system. This can be found under "Admin >> System Analytics". For a user to be able to access these feature, he will need the permission "System Analytics".

Database Analytics

The database analytics page gives you the number of records for all the tables, as well as SQL Statistics since the startup of the application. Additionally you will have the option to take a snapshot of the database while the application is running.

CPU Sampling

The application is taking snapshots of what is going on inside the application. Thanks to that, you can browse the stacktrace of all the methods and see where the application consumes the most time.
The CPU Sampling feature can be found under "Admin >> System Analytics >> CPU Sampling"

Servlet Context Tree

Shows a dump of the applications servlet context handler and all servlets included in the context.

Session Overview

Provides a list of all open sessions in the application instance.

System Properties

Prints the system properties of the application.

Cache Statistics

An overview of various application cache statistics.

Log Configuration

Provides the possibility to change log levels while the application is running.

Metrics

Opens the URL of the OpenMetrics/Prometheus end point in a new tab and prints all the metrics of the application.
