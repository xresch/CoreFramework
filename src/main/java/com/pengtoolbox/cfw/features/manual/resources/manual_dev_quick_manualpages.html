Manual pages are created by adding them to the registry.
Here is an example how this page was added to the manual:

  • Path:  The path consist of the page titles. The titles are delimited with the pipe character, therefore pipe is not allowed in titles.
  • Parents:  Parents that do not exist are automatically created. If you want to have a icon on the page, you need to create them manually as below.
  • Permissions:  Users will only see pages they have permissions for. Permissions will be propagated from child pages to their parents. 
	CFW.Registry.Manual.addManualPage(null, new ManualPage("Administration")
	.faicon("fa fa-cog")
	.addPermission(PERMISSION_ADMIN_MANUAL));
CFW.Registry.Manual.addManualPage("Administration", new ManualPage("Development")
	.faicon("fa fa-code")
	.addPermission(PERMISSION_ADMIN_MANUAL));

CFW.Registry.Manual.addManualPage("Administration | Development", 
		new ManualPage("Create Manual Pages")
			.faicon("fa fa-book")
			.addPermission(PERMISSION_ADMIN_MANUAL)
			.content(HandlingType.JAR_RESOURCE, RESOURCE_PACKAGE, "manual_dev_quick_manualpages.html")
		);

Here another example were you directly add children to a parent page that was already added to the registry. 

	ManualPage dev = CFW.Registry.Manual.addManualPage(null, new ManualPage("Development").faicon("fa fa-code").addPermission(PERMISSION_ADMIN_MANUAL));
dev.addChild(
	new ManualPage("Configuration")
		.faicon("fa fa-cog")
		.addPermission(PERMISSION_ADMIN_MANUAL)
		.content(HandlingType.JAR_RESOURCE, RESOURCE_PACKAGE, "manual_dev_configuration.html")
);

dev.addChild(
	new ManualPage("Permissions")
		.faicon("fa fa-lock")
		.addPermission(PERMISSION_ADMIN_MANUAL)
		.content(HandlingType.JAR_RESOURCE, RESOURCE_PACKAGE, "manual_dev_permissions.html")
	);

dev.addChild(
	new ManualPage("Manual Pages")
		.faicon("fa fa-book")
		.addPermission(PERMISSION_ADMIN_MANUAL)
		.content(HandlingType.JAR_RESOURCE, RESOURCE_PACKAGE, "manual_dev_registry_manualpages.html")
	);

This is an example of how you can add images from a package to your manual page. 
	<img class="img-fluid" src="/cfw/jarresource?pkg=com.pengtoolbox.emp.features.myfeature.resources&file=manual_screenshot.png">

Link to other Manual Pages

This is an example of how you can link to other manual pages without a full page refresh: 
	<a href="#" onclick="cfw_manual_loadPage('Development|Quickstart|Overview');">Overview</a>
