Introduction

The CFWField is a core class of the framework. It basically is an enhanced form of a class field that holds a value of the class instance. Because we create a class for the fields, it is possible to add a lot of commonly needed functions to this fields. This makes a lot of implementations much easier, for example:

  • Creating DB Tables: The column type can be defined on the field, what can be used to automatically create the needed table for saving a field to a Database.
  • ResultSet Mapping:  Values of SQL ResultSets can be mapped using a single function call.
  • Converting to Form:  A collection of CFWFields in a CFWObject can directly be converted to the HTML representation needed for a Form.
  • Mapping Servlet Request Parameters:  You can map servlet request parameters to a collection of CFWFields.
  • Creating DB Queries:  DB Queries can be created through CFWObject and CFWSQL and converted to a variety of different representations, like a list of CFWObejcts or a JSON string.
  • Validation:  Default and custom Validators can directly be added to the field.
  • Autocomplete:  Default customized Autocomplete functionality can directly be added to the field.
  • API:  An API can be easily created through the CFWObject.

CFWField Examples

Here various examples of various CFWField definitions. Create new CFWFields using CFWField.new*() methods. You can set the default value for the field using setValue(). If this is not specified it defaults to null.

Boolean Field with Default true

Creates a simple Boolean field. A BooleanValidator is already added by CFWField.newBoolean(). Sets the default to true. 

private CFWField<Boolean> isDeletable = 
	CFWField.newBoolean(FormFieldType.BOOLEAN, 
						GroupFields.IS_DELETABLE.toString())
			.setValue(true);

Unique Text Field with Validator

Creates a text field with a unique constraint on the database column and a length validator. Default the default column definition set by newString() is "VARCHAR". 

private CFWField<String> name = 
	CFWField.newString(FormFieldType.TEXT, 
					   GroupFields.NAME.toString())
			.setColumnDefinition("VARCHAR(255) UNIQUE")
			.addValidator(new LengthValidator(1, 255));

Password Field

Creates a password field. Password fields will be treated special, in case the value of the field is not null, a dummy value will be put into the form. This will prevent that the actual password can be extracted from the HTML source. 

private CFWField<String> dbPassword = 
	CFWField.newString(FormFieldType.PASSWORD, DBFields.DB_PASSWORD)
			.setDescription("The password of the DB user.");

TextArea as CLOB with 2MB Limit

Creates a textarea and that will be saved to a CLOB field. Limits the data amount to 2MB with the length validator. 

private CFWField<String> description = 
	CFWField.newString(FormFieldType.TEXTAREA, 
					   GroupFields.DESCRIPTION.toString())
			.setColumnDefinition("CLOB")
			.addValidator(new LengthValidator(-1, 2000000));

Number with custom Label

private CFWField<Integer> number =

Select Field with Description

Creates a selection box with the two options "Active" and "Inactive". Adds a description that will be displayed as a tooltip on an info icon in the form in front of the label. 

private CFWField<String> status = 
	CFWField.newString(FormFieldType.SELECT, UserDBFields.STATUS.toString())

			.setDescription("Active users can login, inactive "+
 					 "users are prohibited to login.")

Date Field with Default

Creates a new date picker field with a default epoch time value for "26.01.2020". Times will always be converted to epoch time for request parameters and saving to database. Dates before 1582 are not supported well because of the cutoff between the Julian and Gregorian calendars.( ISO-8601 standard supported) 

private CFWField<Date> date = 
	CFWField.newDate(FormFieldType.DATEPICKER, "DATE")
			.setValue(new Date(1580053600000L));

WYSIWYG Editor

Creates a WYSIWYG editor with an initial value. Please note that you need to use .allowHTML(true) so the user input will not be sanitized.

Default Autocomplete

Creates a text field with a default autocomplete handler. The CFWAutocompleteHandlerDefault will start searching from a single character. It creates a query on the table searching with a LIKE "%searchValue%" statement through the column of the associated field. By default it limits the number of results to 10, what can be adjusted through the constructor or with setMasResults(). 

private CFWField<String> title = CFWField.newString(FormFieldType.TEXT, QuoteFields.QUOTE_TITLE)
		.setDescription("The title of the quote.")
		.addValidator(new LengthValidator(3, 512))
		.setAutocompleteHandler( new CFWAutocompleteHandlerDefault(this.getClass(), 5));

Tags Field As String with Custom Autocomplete

The following creates a tags field that will convert the tags to a comma separated string of tags. It has a custom autocomplete handler that returns a maximum of 5 results. The LinkedHashMap returned will contain the values as "key/label" pairs. The default amount of tags you can set is 128. 

private CFWField<String> tags = CFWField.newString(FormFieldType.TAGS, "TAGS")
		.setValue("foo,test,bar,bla")
		.setAutocompleteHandler(new CFWAutocompleteHandler(5) {
			public LinkedHashMap<Object, Object> getAutocompleteData(HttpServletRequest request, String inputValue) {
				LinkedHashMap<Object, Object>  array = new LinkedHashMap<Object, Object>() ;
				for(int i = 0; i < this.getMaxResults(); i++ ) {
					String tag = "Tag_"+inputValue+"_"+i;
					array.put(tag, tag);
				}
				return array;
			}
		});

Tags As Array Autocomplete from DB wit maximum 20 Tags

The following creates a tags field that will convert the tags to an object array and saves it as well to an ARRAY column to the database. The autocomplete will get the Tags from the DB interface. The use case this example was fetched from has the tags cached internally to reduce query executions. 

private CFWField<Object[]> tags = CFWField.newArray(FormFieldType.TAGS, QuoteFields.TAGS)
		.setDescription("The tags for this quote.")
		.addAttribute("maxTags", "20")
		.setAutocompleteHandler( new CFWAutocompleteHandler(10) {
			@Override
			public LinkedHashMap<Object, Object> getAutocompleteData(HttpServletRequest request, String inputValue) {
				// Find existing tags in DB
				TreeSet<String> tagsSet = OM.DB.Quotes.getTags();
				
				LinkedHashMap<Object, Object> suggestions = new LinkedHashMap<Object, Object>();
				for(String tag : tagsSet) {
					if(tag.toLowerCase().contains(inputValue.toLowerCase())) {
						suggestions.put(tag, tag);
						if(suggestions.size() >= this.getMaxResults()) {
							break;
						}
					}
				}
				return suggestions;
			}
		});

Advanced Tag Selector

With the tag selector you can provide a field where the user can enter a search string and select an object from a database. The selector will use a label for the tag, but will hold a key behind the tag, like an ID.
This is useful for example to select a User by name from the database but getting the ID of the user for processing. Tag selector values will be converted to a LinkedHashMap<String,String> with "key/label" combinations. When saved to the DB it will be converted to a JSON string, why the fieldname has to start with "JSON". To create a tags selector field always use CFWField.newTagsSelector();
The following example shows an example on how to create a tags selector for users. 

private CFWField<LinkedHashMap<String,String>> shareWithUsers = 
	CFWField.newTagsSelector(DashboardFields.JSON_SHARE_WITH_USERS)
		.setLabel("Share with Users")
		.setDescription("Share this dashboard only with specific users.")
		.addAttribute("maxTags", "50")
		.setValue(null)
		.setAutocompleteHandler(new CFWAutocompleteHandler(10) {
			public LinkedHashMap<Object, Object> getAutocompleteData(HttpServletRequest request, String searchValue) {
				return CFW.DB.Users.autocompleteUser(searchValue, this.getMaxResults());					
			}
		});

public static LinkedHashMap<Object, Object> autocompleteUser(String searchValue, int maxResults) {
	
	if(Strings.isNullOrEmpty(searchValue)) {
		return new LinkedHashMap<Object, Object>();
	}
	
	return new User()
		.queryCache(CFWDBUser.class, "autocompleteUser")
		.select(UserFields.PK_ID.toString(),
				UserFields.USERNAME.toString())
		.whereLike(UserFields.USERNAME.toString(), "%"+searchValue+"%")
		.and().not().is(UserFields.PK_ID, CFW.Context.Request.getUser().id())
		.limit(maxResults)
		.getAsLinkedHashMap(UserFields.PK_ID.toString(), 
							UserFields.USERNAME.toString());
	
}
